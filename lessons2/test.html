Flex容器项目属性详解

Flexbox布局提供了多种属性来控制Flex项目(子元素)的行为。这些属性直接应用于Flex容器内的项目，允许您精确控制每个项目的布局方式。

1. order属性

order属性控制Flex项目的排列顺序。默认情况下，所有项目的order值为0，项目按照它们在HTML中出现的顺序排列。

.item {
  order: <integer>; /* 默认是0 */
}
1 (order: 0)
2 (order: 0)
3 (order: 0)
4 (order: 0)

使用场景： 当需要改变项目的视觉顺序而不修改DOM结构时非常有用。例如响应式布局中，不同屏幕尺寸下可能需要不同的项目顺序。
注意：虽然order可以改变视觉顺序，但不会影响DOM顺序，因此对于屏幕阅读器和键盘导航的顺序不会改变，可能会导致可访问性问题。

2. flex-grow属性

flex-grow定义项目放大的能力，即在容器有剩余空间时，项目应该如何分配这些空间。

  flex-grow: <number>; /* 默认是0 */
}

注意： flex-grow指定的是项目的放大比例，不是最终大小的比例。实际大小还取决于flex-basis和其他因素。
使用场景： 创建适应不同屏幕尺寸的流体布局，如导航菜单中某些项目需要占据更多空间。

3. flex-shrink属性

flex-shrink定义项目的缩小能力，即在空间不足时，项目应该如何缩小。

使用场景： 防止某些重要内容在空间不足时被压缩，如表格中的某些列需要保持宽度。
注意： flex-shrink: 0的项目可能会超出容器，导致溢出，需要配合overflow属性使用。

4. flex-basis属性

flex-basis定义了项目在分配多余空间之前的默认大小。

.item {
  flex-basis: auto | <width>; /* 默认是auto */
}

使用场景： 为项目设置初始大小，在保持比例同时允许根据需要增长或收缩。
注意： flex-basis会覆盖width属性(如果主轴是水平的)。设置min-width或max-width可以限制最终的弹性尺寸。

5. flex属性（简写形式）

flex是flex-grow、flex-shrink和flex-basis的简写属性，推荐使用这个属性而不是单独写三个分离的属性。

.item {
  flex: flex-grow flex-shrink flex-basis;
}

6. align-self属性

align-self允许单独的对某个Flex项目进行交叉轴（纵轴）方向上的对齐方式设置，会覆盖align-items的属性。

.item {
  align-self: auto | flex-start | flex-end | center | baseline | stretch;
}
默认
flex-start
center
flex-end
stretch

align-self: flex-start

项目位于容器交叉轴起点

start

align-self: center

项目位于容器交叉轴中心

center

align-self: flex-end

项目位于容器交叉轴末端

end

align-self: stretch

项目拉伸填满容器交叉轴

stretch
使用场景： 当容器内大多数项目使用一种对齐方式，但个别项目需要特殊对齐时特别有用。
注意： align-self的值会覆盖容器的align-items值。如果flex-direction是column，则align-self控制的是水平方向的对齐。
